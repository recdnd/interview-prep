# 小工具傻瓜說明＋接球包補強提案

這份是超精簡版，給「臨上場前 30 秒看一眼」用。

---

## 1) 右邊三個小工具（由上到下）

1. `CCE`（最上）  
   你要**自己講一段**時用。會給你 `[起手] [中段] [収束]` 三段骨架。

2. `CRE`（中間）  
   不想思考要用 CCE 還是 CBL 時用。它會在「獨白」和「接球」之間切換。

3. `CBL`（最下）  
   對方講完後你要**接球再拉回自己**時用。會給你 `[接球] [微肯定] [軽い反問] [つなぎ]`。

---

## 2) 按鈕是什麼意思（白話）

## `CCE` 的 `A / G / UX / SYS`

- `A` = Auto，自動猜模式（看現在題目標題）
- `G` = General，一般通用講法
- `UX` = 偏使用者體驗、可理解性
- `SYS` = 偏系統、結構、狀態、取捨

## `CRE` 的 `A / S / R`

- `A` = Auto，自動決定
- `S` = Speak，強制走「獨白三段」（等同 CCE 路線）
- `R` = Relay，強制走「接球四段」（等同 CBL 路線）

## `CBL` 的 `A / 流 / 構 / 現 / 明`

- `A` = Auto，自動選 AFF 類型
- `流` = process（流程順、講法順）
- `構` = structure（結構清楚、切分合理）
- `現` = reality（貼近實務、運用、現場）
- `明` = clarity（表達清楚、焦點明確）

一句話記法：  
`CCE` 管你「怎麼講完一段」，`CBL` 管你「怎麼接住對方」，`CRE` 幫你自動二選一。

## 3) 哪些 pack 比較缺「接球語」？（先補哪裡）

先補這 6 個最有感：

1. `coding-edge`
2. `coding-test`
3. `coding-test-loglass`
4. `coding-test-loglass`（含原 lite 極短題，`〔極短〕` 接頭）
5. `product-engineer-cheatsheet`
6. `ml-research-pack`

原因很簡單：這幾包多是「完整答題」，比較少「先接一下再回答」的句型。

---

## 4) 接球語要從哪裡抓？

可以，優先從這兩個來源抓（你問的「從這個包裡找」就用這路線）：

- 來源 A：`CBL` 現有池（最穩）
  - `CBL.ack`（接球）
  - `AFF.*`（微肯定）
  - `CBL.expand`（接回自己）
- 來源 B：`technical-ball` 內偏口語、可承接的句子（次穩）

建議原則：  
**不要整段搬**，只抽「半句～一句」當前綴，接到原題答案前面。

## 5) 實作提案（最省事版本）

每個「缺接球」pack 都加 2 題就夠：

1. `短句工具 / 接球前綴（軽）`
2. `短句工具 / 軟收尾（軽）`

### A. 接球前綴（可直接共用）

- `そうですね。`
- `今のお話、すごく分かりやすいです。`
- `その観点、かなり重要だと思います。`
- `自分の感覚でも、その見方は近いです。`
- `まず前提としては、その理解で考えています。`

### B. 軟收尾（可直接共用）

- `この辺りは、実務だともう少し現実的に見ると思います。`
- `まだ改善の余地はあると思っています。`
- `現時点では、まずそのように整理しています。`
- `必要であれば、もう少し具体化してお話しできます。`
- `御社だとこの点はどう判断されることが多いですか。`
