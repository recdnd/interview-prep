# david-pich-routing

給 **Cursor／自己** 用的「整場控制器」：**題目與 `packs/david-pich.js` 的 `id` 一致**（2026 改版：16 題、逆質問・判定・デモ・締めまで一本線）。

---

## 目標

- 開場は**誇りではなく相手の立ち位置の言語化**（記事理解の返し）。
- 中盤は**自分語りより相手の観察サンプル化**（DS→SWE、Loglass 現場、泥度）。
- 終盤は**判定を相手に言語化させる**＋必要なら **id 15** でデモ供与、**id 16** で「今日の収穫」締め。

---

## 全局 flow（建議）

`START`  
→ **id 1**（入り）→ **id 2**（相手をサンプル化）→ **id 3**（接点）→ **id 4**（自己＝職能）  
→ **id 5〜8**（Loglass／DS 転換／泥度／判断軸）※相手のターンを多めに  
→ **id 9**（Ruvia）→ **id 10**（AI）→ **id 11**（具体フロー）  
→ **id 12〜14**（逆質問・伸び方・見立て）  
→（必要時）**id 15** デモ → **id 16** 締め  
→（時間が残る／相手がチーム像を聞きたいとき）**id 17〜22** で協働イメージを短く足す → **id 23〜24** で Loglass 文脈に接続  

**後半（id 5 以降で相手が長く話すとき）**：pack を増やしすぎず **CBL／CRE relay** で短文接球。

## cross-pack routing

### Spiral を聞かれたら

- 統合後 **`loglass`**：題名接頭 **`〔15s〕`** の Spiral 動機系（旧 frequent 相当）→ 続けて **`〔語彙〕`** 内の Spiral／翻訳まわり（旧 internal-lexicon 相当）を当てる  
- **avoid**：ontology 長講義。

### 技術／研究が続くとき

- `ml-research-pack`（**問い返し**）。

### 小トーク・軽い沈黙

- **CBL**、**CCE**。

## 全局注意

### 不要

- 抽象だけ・自分語りだけ・Spiral 独演。  

### 要做

- **id 1〜4** で相手のフレーミングを先に置く。  
- **id 5〜8** で相手のターンを増やす。  
- **id 12〜14** で「一緒に働く想像」を相手の言葉で引き出す。
